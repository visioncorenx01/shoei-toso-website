# ミニファイの方法（初心者向け）

## ミニファイって何？

**ミニファイ** = ホームページのファイル（HTML と CSS）を**圧縮すること**です。

- 改行や余分な空白を削除して、ファイルを小さくします
- 見た目や動きは**まったく変わりません**
- ソースを見たときに「1行にギュッと詰まって読みにくい」形になるので、ちょっとだけコピーされにくくなります

**いつ使う？**  
サイトをサーバーにアップロードする前に、ミニファイしたファイル（**dist** フォルダの中身）をアップロードすると、本番用として使えます。

---

## 準備：Node.js が入っているか確認する

ミニファイには **Node.js**（ノード・ジェイエス）というソフトが必要です。

### 確認のしかた

1. **ターミナル**を開く  
   - Mac： Spotlight（⌘ + スペース）で「ターミナル」と入力して起動  
   - Windows： 「スタート」→「cmd」または「PowerShell」で「コマンドプロンプト」や「ターミナル」を開く

2. 次の文字を**そのまま**入力して、**Enter** を押す  
   ```bash
   node -v
   ```

3. 結果の見方  
   - **v18.0.0** のように、`v` のあとに数字が出れば → **入っています**。このガイドの「手順」に進んでください。  
   - **「コマンドが見つかりません」** のようなメッセージ → **入っていません**。  
     → [Node.js の公式サイト](https://nodejs.org/ja/) から「LTS」と書いてある方をダウンロードしてインストールしてから、もう一度 `node -v` を試してください。

---

## 手順（2つのコマンドだけ）

ここからは、**「昇栄塗装ホームページ」のフォルダの中**で作業します。

### ステップ1：フォルダに移動する

ターミナルで、このサイトのフォルダに移動します。

- **Mac でデスクトップにある場合（おすすめ）**  
  ```bash
  cd ~/Desktop/昇栄塗装ホームページ
  ```
  ※ `~` は「ホームフォルダ」の意味なので、どこからでも同じように動きます。

- **Windows の場合**  
  ```bash
  cd Desktop\昇栄塗装ホームページ
  ```
  または  
  ```bash
  cd %USERPROFILE%\Desktop\昇栄塗装ホームページ
  ```

**重要：移動できたか確認してから次へ**  
次のコマンドで、今いる場所を確認します。  
```bash
pwd
```
表示が `/Users/あなたの名前/Desktop/昇栄塗装ホームページ` のような**プロジェクトのパス**になっていればOKです。  
さらに `ls` と入力して **package.json** が一覧に出ていれば、このフォルダで **npm install** を実行して大丈夫です。  
※ ここが違うフォルダのまま **npm install** すると「package.json がない」というエラーになります。

---

### ステップ2：初回だけ「パッケージを入れる」

**初めてミニファイするときだけ**、次のコマンドを実行します。

```bash
npm install
```

- 意味：「このプロジェクトで使うツールを、自動でダウンロードして準備する」
- 数十秒〜1分ほどかかることがあります
- 最後にエラーが出ていなければ成功です
- **2回目以降は、このステップは不要です**

---

### ステップ3：ミニファイを実行する（またはプレビュー）

毎回ミニファイしたいときに、このコマンドを実行します。

```bash
npm run minify
```

- 意味：「index.html と style.css を圧縮して、dist フォルダに出力する」
- 数秒で終わります
- 画面に「✓ index.html をミニファイ」「✓ style.css をミニファイ」などと出れば成功です

**プレビュー（サイトを確認したいとき）**  
同じフォルダで次を実行すると、ブラウザでサイトを確認できます。  
```bash
npm run preview
```
表示された **http://localhost:3000** をブラウザで開いてください。  
※ **npm run preview も「昇栄塗装ホームページ」のフォルダで実行**してください。ホームフォルダ（~）で実行すると「package.json がない」とエラーになります。そのときは先に `cd ~/Desktop/昇栄塗装ホームページ` で移動してから実行してください。

---

終わると、**dist** という名前のフォルダができています（もともとある場合は中身が上書きされます）。

| dist の中身 | 説明 |
|-------------|------|
| index.html  | 圧縮されたトップページ |
| style.css   | 圧縮されたスタイル |
| sitemap.xml | サイトマップ（そのままコピー） |
| images/     | 画像フォルダ（そのままコピー） |

---

## 公開（サーバーにアップロード）するとき

- **アップロードするのは「dist フォルダの中身」だけ**にします
- dist の中の **index.html**、**style.css**、**sitemap.xml**、**images** フォルダを、サーバーの公開用フォルダ（例：www や public_html）にそのままアップロードしてください
- **元の index.html や style.css（dist の外にあるもの）は、編集用として残しておき、直接アップロードしない**のがおすすめです

**流れのイメージ**

1. いつも通り、元の **index.html** や **style.css** を編集する  
2. 公開したいタイミングで **npm run minify** を実行する  
3. できた **dist** の中身だけをサーバーにアップロードする  

---

## ミニファイで実際にやっていること

| ファイル | やっていること |
|---------|----------------|
| HTML    | 改行・余白の削除、コメント削除。表示やリンクの動きはそのまま |
| CSS     | 改行・余白の削除。デザインはそのまま |

※ ブラウザで見た見た目や動作は変わりません。ファイルが小さくなり、ソースが読みにくい形になるだけです。

---

## 注意しておくこと

- **dist フォルダの中は、毎回「npm run minify」で上書きされます**  
  → dist の中のファイルを直接編集しないでください。編集するのはいつも「元の index.html / style.css」です。

- **中身を変えたら、もう一度ミニファイしてからアップロード**  
  → 元の HTML や CSS を直したあとは、もう一度 **npm run minify** を実行し、新しくできた dist の中身をアップロードし直してください。

---

## うまくいかないとき

- **「npm が見つかりません」**  
  → Node.js が入っていないか、ターミナルを開き直す必要があるかもしれません。上の「準備」のところで `node -v` を再度確認してください。

- **「そのようなファイルやディレクトリはありません」**  
  → `cd` で「昇栄塗装ホームページ」のフォルダに正しく移動できていない可能性があります。今いるフォルダは `cd` だけ入力して Enter で確認できます（Mac では `pwd` でも可）。

- **npm install でエラーが出る**  
  → インターネットに接続されているか確認してください。会社や学校のネットで「npm が使えない」場合もあります。

---

このガイドの手順どおりに「npm install」→「npm run minify」までできれば、ミニファイは完了です。公開時は、**dist の中身だけ**をアップロードするようにしてください。
