# ブログ機能セットアップ手順（昇栄塗装ホームページ）

このサイトのブログは **microCMS（マイクロCMS）** という無料のサービスで記事を書き、
**Netlify（ネットリファイ）** が自動でホームページに反映する仕組みです。

```
[スマホ/PCで記事を書く]      [自動でビルド]            [ホームページに反映]
   microCMS  ──公開ボタン──▶  Netlify ──記事を取得して──▶ blog/ に静的ページ生成
                （Webhook）          HTML化               （SEOに強い形）
```

一度この手順で設定すれば、**あとはスマホで記事を書いて「公開」を押すだけ**でホームページが自動更新されます。
パソコンでの作業（コマンド実行やアップロード）は不要です。

---

## 全体の流れ（やることリスト）

1. microCMS のアカウントを作る
2. microCMS で「ブログ用のAPI（`blogs`）」を作る
3. microCMS の「APIキー」をコピーする
4. Netlify に環境変数（APIキーなど）を登録する
5. Netlify の「Build hook（ビルド用のURL）」を作る
6. microCMS の「Webhook」に、そのURLを登録する
7. テスト投稿して、自動で反映されるか確認する

> ※ Netlify とこのサイトのソース（GitHub などのリポジトリ）が連携済みであることが前提です。
> まだ「手動アップロード（Netlify Drop）」で運用している場合は、先に「補足B」を読んでください。

---

## 1. microCMS のアカウントを作る

1. [https://microcms.io/](https://microcms.io/) を開く
2. 「無料ではじめる」からアカウントを作成（メールアドレス／Googleなど）
3. 案内に従って「サービス」を作成します
   - **サービス名**：例「昇栄塗装ブログ」など分かりやすい名前
   - **サービスID**：例 `shoei-toso`（半角英数字。URLの一部になります）
   - ここで決めた **サービスID** が、あとで使う「サービスドメイン」です
     （サービスのURLが `https://shoei-toso.microcms.io` なら、ドメインは `shoei-toso`）

---

## 2. ブログ用のAPI（`blogs`）を作る

microCMS の管理画面で「API」を作成します。**設定どおりに作るのが大事**です。

1. 管理画面で「**コンテンツ（API）を作成**」を選ぶ
2. 作り方は「**一から作成する**」を選ぶ
3. 基本情報を入力
   - **API名**：`ブログ`（表示名。なんでもOK）
   - **エンドポイント（API ID）**：必ず **`blogs`**（半角小文字）にする ← 重要
4. APIの型は「**リスト形式**」を選ぶ
5. 「APIスキーマ（入力フィールド）」を次のとおり作成する

| フィールドID | 表示名 | 種類 | 必須 | 備考 |
|---|---|---|---|---|
| `title` | タイトル | テキストフィールド | ○ | 記事の見出し |
| `content` | 本文 | リッチエディタ | ○ | 文章・画像・見出しなど |
| `category` | カテゴリ | セレクト | 任意 | 選択肢に「お知らせ」「施工事例」「コラム」などを追加 |
| `eyecatch` | アイキャッチ画像 | 画像 | 任意 | 記事のサムネイル |

> - **フィールドIDは上の表のとおり**（`title` / `content` / `category` / `eyecatch`）にしてください。
>   これがホームページ側のプログラムと一致している必要があります。
> - **公開日**は microCMS が自動で持っている「publishedAt（公開日時）」を使うので、自分で作る必要はありません。
> - `category` のセレクトは「**複数選択しない**（単一選択）」でも「複数選択」でも、どちらでも動きます。

6. 「作成」を押して完了

---

## 3. APIキーをコピーする

1. microCMS 管理画面の左下「**サービス設定**」→「**APIキー**」を開く
2. 表示されている **APIキー（デフォルトキー）** をコピーします
   - 権限は「**GET（取得）が有効**」になっていればOKです
3. 同じく「**サービスドメイン**」（手順1で決めたサービスID。例 `shoei-toso`）も控えておきます

この2つを次の手順で Netlify に登録します。
- **サービスドメイン**（例 `shoei-toso`）
- **APIキー**（長い英数字の文字列）

> ⚠ APIキーは「パスワード」と同じです。人に見せたり、SNS等に貼らないでください。

---

## 4. Netlify に環境変数を登録する

1. [https://app.netlify.com/](https://app.netlify.com/) にログイン
2. このサイトを選ぶ
3. 上部メニュー「**Site configuration（サイト設定）**」→「**Environment variables（環境変数）**」を開く
4. 「**Add a variable**」→「**Add a single variable**」で、次の2つを登録する

| Key（名前） | Value（値） |
|---|---|
| `MICROCMS_SERVICE_DOMAIN` | 手順3のサービスドメイン（例 `shoei-toso`） |
| `MICROCMS_API_KEY` | 手順3のAPIキー |

5. 保存する

> 名前（Key）は**スペルも大文字小文字も正確に**入力してください。1文字でも違うと動きません。

---

## 5. Netlify の Build hook（ビルド用URL）を作る

「このURLにアクセスがあったら、サイトを作り直す（ビルドする）」というURLを作ります。

1. Netlify のサイト設定「**Build & deploy**」→「**Build hooks**」を開く
2. 「**Add build hook**」を押す
   - **名前**：例 `microcms-publish`
   - **ブランチ**：通常は `main`（公開に使っているブランチ）
3. 作成すると、`https://api.netlify.com/build_hooks/xxxxxxxx` のような **URL** が表示される
4. この **URLをコピー**しておく（次の手順で使います）

---

## 6. microCMS の Webhook に、そのURLを登録する

記事を「公開」したときに、手順5のURLが自動で叩かれるようにします。

1. microCMS で `blogs` のAPI設定（API設定 → **Webhook**）を開く
2. 「**追加**」→「**カスタム通知（または「その他」/「Webhook」）**」を選ぶ
3. **URL** に、手順5でコピーした Netlify の Build hook URL を貼り付ける
4. 通知のタイミングは、少なくとも次にチェックを入れる
   - コンテンツの **公開・更新時**
   - コンテンツの **削除時**（消したらサイトからも消えるように）
5. 保存する

これで「microCMSで公開 → Netlifyが自動でビルド → サイトに反映」がつながりました。

---

## 7. テスト投稿して確認する

1. microCMS で `blogs` に新しいコンテンツを1件作る
   - タイトル・本文・カテゴリを入力（アイキャッチ画像は任意）
2. 「**公開**」ボタンを押す
3. 1〜3分ほど待つ（Netlify がビルドする時間）
4. ホームページを開いて確認
   - トップページの「**ブログ・お知らせ**」に最新記事が出ているか
   - 「ブログ一覧を見る」→ 一覧ページ → 記事をタップ → 記事ページが開くか

反映されない場合は「うまくいかないとき」を見てください。

---

## スマホでの運用フロー（設定後の毎日の使い方）

1. スマホで microCMS の管理画面にログイン（ブラウザでOK。ブックマーク推奨）
2. `blogs` →「追加」で記事を書く
   - タイトル、本文（写真も入れられます）、カテゴリを選ぶ
3. 「**公開**」を押す
4. **これだけ**。数分でホームページに自動で反映されます

> 修正したいときも、記事を直して「公開（更新）」を押すだけ。削除も同様に自動反映されます。

## 補足A：パソコンでローカル確認したいとき（任意・上級者向け）

パソコンにこのプロジェクトがある場合、手元で表示確認できます。

```bash
cd ~/Desktop/昇栄塗装ホームページ
npm install          # 初回だけ
npm run build-blog   # 記事を取得してブログページを生成（microCMS未設定でも落ちません）
npm run preview      # http://localhost:3000 で確認
```

- `.env` ファイルに次のように書いておくと、ローカルでも microCMS から記事を取得します。
  （`.env` は秘密情報なので **絶対に共有・コミットしないでください**）

```
MICROCMS_SERVICE_DOMAIN=shoei-toso
MICROCMS_API_KEY=（あなたのAPIキー）
```

- microCMS を設定していない（または取得に失敗した）場合は、
  プロジェクト内の `blog-data.json`（サンプル記事）を使って表示確認できます。
  これは動作確認用のサンプルなので、本番では microCMS の記事が優先されます。

## 補足B：今の公開方法に合わせた設定

このサイトは `npm run minify` で **`dist/` フォルダ**に本番用ファイルを書き出す構成です。
そのため、`netlify.toml`（自動で用意済み）は次のようになっています。

```toml
[build]
  command = "npm run build"   # = ブログ生成（build-blog）＋ ミニファイ（minify）
  publish = "dist"            # dist フォルダの中身を公開
```

- **GitHub等のリポジトリと Netlify が連携している場合**：上記のままでOKです。
  microCMS の公開 → Build hook → Netlify が `npm run build` を実行 → `dist/` を公開、と自動で流れます。

- **今は手動アップロード（Netlify Drop でzipをドロップ）している場合**：
  まずは「リポジトリ連携」に切り替えると、この自動化（手順5・6）が使えます。
  切り替えが難しい場合は、これまでどおり手元で `npm run build` を実行し、
  できた `netlify-deploy.zip`（または `dist/` の中身）を Netlify にアップロードしてください。
  この場合は手順5・6（自動化）は使えませんが、ブログ自体は同じように表示されます。

- **「ルート公開」にしたい場合**（dist を使わず直下を公開）：
  `netlify.toml` の `publish` を `"."`、`command` を `"npm run build-blog"` に変更してください。

## うまくいかないとき

- **記事が反映されない**
  - Netlify の「Deploys（デプロイ履歴）」を見て、ビルドが走っているか／成功しているか確認
  - microCMS の Webhook の URL が、Netlify の Build hook の URL と一致しているか確認
  - 環境変数 `MICROCMS_SERVICE_DOMAIN` / `MICROCMS_API_KEY` の名前・値が正しいか確認

- **ビルドは成功するのに記事が空（0件）**
  - APIキーの権限に「GET」があるか
  - API ID が `blogs`（小文字）になっているか
  - 記事が「公開」状態か（下書きのままになっていないか）

- **画像が出ない**
  - microCMS の `eyecatch`（画像フィールド）にアップロードされているか確認

困ったときは、Netlify の「Deploys」→ 失敗したビルドのログ（赤い文字）を見ると原因のヒントが出ます。
